# Lab 25: Pattern Integration

Ця лабораторна робота демонструє взаємодію чотирьох фундаментальних патернів проектування:

1.  **Singleton (LoggerManager):** Забезпечує єдину точку доступу до системи логування в усьому додатку.
2.  **Factory Method (LoggerFactory):** Дозволяє динамічно змінювати спосіб логування (Console або File), не змінюючи код клієнта.
3.  **Strategy (IDataProcessorStrategy):** Дає можливість змінювати алгоритм обробки даних (шифрування або стиснення) під час виконання програми.
4.  **Observer (DataPublisher):** Реалізує механізм сповіщення логера про те, що дані були успішно оброблені.

## Результати виконання сценаріїв
* **Сценарій 1:** Продемонстровано базовий цикл: обробка (Encrypt) -> подія -> логування в консоль.
* **Сценарій 2:** Зміна `LoggerFactory` на `FileLoggerFactory` перенаправила логування в файл `log.txt`.
* **Сценарій 3:** Зміна стратегії в `DataContext` змінила метод обробки даних на `Compress`
